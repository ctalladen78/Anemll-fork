# TurboQuant & ANE Chat

This project benchmarks and optimizes LLM inference on Apple Silicon using **TurboQuant** KV cache quantization and **ANE** (Apple Neural Engine) acceleration.

## TurboQuant

TurboQuant is a KV cache quantization technique that dramatically reduces memory usage while maintaining performance:

| Configuration | KV Cache (8k) | KV Cache (32k) | Prefill Speed | Generation |
| :--- | :--- | :--- | :--- | :--- |
| Standard (FP16) | ~160 MiB | ~640 MiB | 710 t/s | 31.5 t/s |
| **Asymmetric (Q8/T4)** | **42.6 MiB** | **170.5 MiB** | **712 t/s** | **32.3 t/s** |

**Key Finding**: Asymmetric TurboQuant (`-ctk q8_0 -ctv turbo4`) delivers **3.7x smaller KV cache** with **slightly better throughput** than standard inference.

### Optimal Server Command

```bash
./llama-cpp-turboquant/build/bin/llama-server \
  -m <model.gguf> \
  -ctk q8_0 \
  -ctv turbo4 \
  --flash-attn \
  -c 32768 \
  --port 8080
```

## Results

- **32k context** fits in ~170 MiB KV cache (vs ~640 MiB standard)
- **Asymmetric mode** is strictly superior for general use
- Thermal throttling can reduce throughput by ~38% - allow cooling time between heavy benchmarks
