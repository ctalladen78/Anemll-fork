# TurboQuant Performance Analysis: Qwen3.5-35B-A3B

This report summarizes the benchmarking results for the `Qwen3.5-35B-A3B` Mixture-of-Experts (MoE) model on Apple Silicon (M5 Architecture) using the `llama-cpp-turboquant` fork.

## KV Cache Architecture Insights

The `Qwen3.5-35B-A3B` model utilizes a hybrid architecture:
- **Total Layers**: 40
- **Attention Layers**: 10 (1 every 4 layers)
- **SSM/Recurrent Layers**: 30
- **Heads (KV)**: 2
- **Head Dim**: 256

> [!NOTE]
> Because only 25% of layers utilize a standard KV cache, the memory footprint for high context is significantly lower than a dense model of equivalent size.

## 5. Final Recommendation

For **Qwen3.5-35B-A3B** on Apple Silicon, use the following server flags for optimal performance/memory balance:

```bash
./llama-server -m models/qwen35-35b-moe.gguf -ctk q8_0 -ctv turbo4 --flash-attn
```

- **Why?**: The `Asymmetric` mode (Q8/Turbo4) provides the highest prefill throughput while maintaining a negligible generation speed penalty (0.6 t/s).
- **Context Window**: 32k context is safely supported in less than 200 MiB of VRAM footprint, leaving ample room for the 20.49 GiB model weights.
